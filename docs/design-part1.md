# 我是如何设计 Space 的（上篇）：产品设计

## AI 记忆的困境

ChatGPT、Gemini、Claude 是现在 AI 的"御三家"。它们都提供了记忆机制——在对话过程中，AI 会自动将重要内容记录到全局记忆库，下次对话时加载，这样 AI 就能记得你提到过的事情。

但这造成了一个问题：**串味**。

你在任何一个对话窗口都会读取到所有记忆，也会将内容记入同一个全局记忆。可是我们的生活是分不同领域、不同类别的——工作中的事并不想在聊生活时被提起；有时候想感性地倾诉，有时候想理性地提问，全局记忆把这些全搅在一起了。

于是 ChatGPT 和 Claude 推出了 Project 功能，同一个 Project 的指示、记忆、文件只在 Project 内共享。这确实解决了隔离的问题。

但 Project 本质上还是一个**文件夹**，不是一个**空间**。

在 Project 中，对话之间本质上还是独立的。虽然新对话可以从历史记录中检索出一些信息，但这是零碎的，不是整体的；是临时的，不是持久的。在一个 Project 里聊了三个月，它不会比第一天更了解你。

## 我们到底在用 AI 做什么

回到一个根本问题：我们用 AI 在用什么？

我觉得大致分三种：

**把 AI 当工具**——一次性的事：翻译、查资料、做表格。用完即走，不需要记忆。

**把 AI 当搭档**——和 AI 一起讨论，汲取灵感，产出内容。这需要 AI 记得上下文和你的偏好。

**把 AI 当树洞**——用 AI 陪伴，写日记、聊困惑、分析梦、倾诉情绪。AI 是一片心灵的空间，承载你的想法与情绪。

后两种用法，我们需要的不只是"记忆"。更确切地说，我们希望营造一个个**空间**，在不同的空间内，AI 能记得我们是谁，聊过什么，这个空间的氛围是什么，是什么样的节奏，与默契。

## 空间的隐喻

人的生活本来就是由不同的空间组成的。走进不同的地方，自然地想不同的事、说不同的话、成为不同状态的自己。空间给你一个暗示：在这里，你可以是这样的。

和 AI 的对话也是一样。你关于梦的对话，聊了三个月之后，有了自己的语言、自己的节奏、自己的积累。它值得有一个自己的地方待着，被好好保存，下次你走进去的时候一切都还在。

我想以梦的空间为例：

> 我经常和 AI 聊梦。如果扔进普通的全局对话框，AI 会像一个尽职尽责的解梦字典。它可能会生硬地调用一套通用的心理学理论——用荣格的原型去套用我的梦境，或者用拉康的镜像阶段做一套标准化的精神分析。它很聪明，但像一个初次见面的、穿着白大褂的医生，试图用理性的手术刀切割我感性的潜意识。
>
> 但在一个专属的"梦的 Space"里，体验截然不同。在这里聊了三个月之后，这个空间不再需要我从头解释背景，也不需要我刻意设定 prompt。它开始熟悉我潜意识的语言和独有的意象系统。它不是在做刻板的精神分析，而是像一位同行的老友。那些深奥的心理学框架退居幕后，真正浮现出来的，是"我"独一无二的心理轨迹。
>
> 这种默契，是随着时间在这个特定的"场域"中**生长**出来的。

于是，当我们想聊一个领域的事情时，打开 AI，进入一个空间。这个空间内，是一种特定的感觉，新建一个空间内的对话，一切都是空间内的氛围。

## 记忆如何生长

Space 的记忆系统不是全局记忆那种"记录关键信息"的方式，也不是 RAG 那种"检索相关片段"的方式。它是一种**认知蒸馏**。

每个 Space 有一组 Context 文档。每个文档覆盖一个维度，是正交的。比如在梦的空间里，可能有：

- `dreamer.md`——用户是谁，潜意识中反复出现的主题与情结
- `symbols.md`——用户独有的意象词典，水、门、坠落在 ta 的梦里分别意味着什么

这些 Context 不是用户手写的 prompt，而是空间自己**长出来的**理解——从一轮轮对话中蒸馏而成。每次新对话开始时，AI 读取这些认知文档，带着对你的理解开始交流。

而 SPACE.md 是 AI 用第一人称写的空间自述：这是一个什么样的空间，什么氛围，什么节奏，什么默契，如何与用户相处。它和 Context 一起构成了空间的"人格"。

## 归档：一个命令背后的仪式感

对话结束后，用户输入 `/archive`，一个命令。背后发生的事情是：

1. **生成记录**：将当前对话提炼为一份摘要——聊了什么、产生了什么、情绪的走向、关键的转折。保存到 records 目录，以日期+标题命名。
2. **更新认知**：AI 审视这轮对话，判断是否有新的认知需要沉淀。如果发现了新的意象、新的偏好、新的主题模式，就更新已有的 Context 文档，或创建新的。如果没有新东西，就不动。这一步是 Space 生长的核心机制。
3. **更新空间描述**：更新 SPACE.md 中的 Context 索引，如果有值得记录到空间描述中的内容，就更新。
4. **保存原始对话**：完整的对话以 JSONL 格式存档，未来可以回溯。
5. **清空当前对话**：归档完成，对话窗口清空，回到 Space 的初始状态——Context 已更新，准备好迎接下一轮对话。

前三步都需要用户手动确认。AI 不能偷偷改你的记忆。

整个流程对用户来说就是一个 `/archive`。背后发生的是：经历变成记录，记录变成认知，认知融入空间。

## 交互：类 Claude Code 的终端体验

Space 的交互是终端界面，类似 Claude Code 的风格——底部输入框，消息向上滚动，状态栏显示 token 用量和当前模型。

默认就是普通聊天，不属于任何空间。用 `/space <名字>` 进入一个空间，用 `/archive` 归档，用 `/new` 开始新对话，用 `/exit` 退出回到普通聊天。

极简的命令式交互，没有多余的 UI 元素。你始终知道自己在哪个空间。
